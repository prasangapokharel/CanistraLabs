=== SUPPORT (--help) ===
Transfer ICP from the user to the destination account identifier or principal

Usage: dfx ledger transfer [OPTIONS] --memo <MEMO> [TO]

Arguments:
  [TO]  AccountIdentifier of transfer destination

Options:
      --network <NETWORK>
          Override the compute network to connect to. By default, the local network is used. A valid
          URL (starting with `http:` or `https:`) can be used here, and a special ephemeral network
          will be created specifically for this request. E.g. "http://localhost:12345/" is a valid
          network name [env: DFX_NETWORK=]
      --to-principal <TO_PRINCIPAL>
          Principal of transfer destination
  -v, --verbose...
          Displays detailed information about operations. -vv will generate a very large number of
          messages and can affect performance
      --playground
          Shorthand for --network=playground. Borrows short-lived canisters on the real IC network
          instead of creating normal canisters
  -q, --quiet...
          Suppresses informational messages. -qq limits to errors only; -qqqq disables them all
      --to-subaccount <TO_SUBACCOUNT>
          Transfer ICP to this subaccount
      --from-subaccount <FROM_SUBACCOUNT>
          Subaccount to transfer from
      --ic
          Shorthand for --network=ic
      --log <LOGMODE>
          The logging mode to use. You can log to stderr, a file, or both [default: stderr]
          [possible values: stderr, tee, file]
      --amount <AMOUNT>
          ICPs to transfer to the destination AccountIdentifier Can be specified as a Decimal with
          the fractional portion up to 8 decimal places i.e. 100.012
      --logfile <LOGFILE>
          The file to log to, if logging to a file (see --logmode)
      --icp <ICP>
          Specify ICP as a whole number, helpful for use in conjunction with `--e8s`
      --identity <IDENTITY>
          The user identity to run this command as. It contains your principal as well as some
          things DFX associates with it like the wallet [env: DFX_IDENTITY=]
      --e8s <E8S>
          Specify e8s as a whole number, helpful for use in conjunction with `--icp`
      --provisional-create-canister-effective-canister-id <PRINCIPAL>
          The effective canister id for provisional canister creation must be a canister id in the
          canister ranges of the subnet on which new canisters should be created
      --memo <MEMO>
          Specify a numeric memo for this transaction
      --fee <FEE>
          Transaction fee, default is 10000 e8s
      --ledger-canister-id <LEDGER_CANISTER_ID>
          Canister ID of the ledger canister
      --created-at-time <CREATED_AT_TIME>
          Transaction timestamp, in nanoseconds, for use in controlling transaction-deduplication,
          default is system-time. //
          https://internetcomputer.org/docs/current/developer-docs/integrations/icrc-1/#transaction-deduplication-
  -h, --help
          Print help
